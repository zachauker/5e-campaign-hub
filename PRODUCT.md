# Product

## Register

product

## Users

Dungeon Masters running live D&D combat sessions. Single user, high-stress context: managing initiative, tracking HP and conditions across 6–12 combatants, referencing stat blocks, and keeping the game moving — all simultaneously. They're sitting at a table with players watching. Every second the DM spends reading the UI is a second the table is waiting. The tool competes with dice rolls, crosstalk, and ambient chaos.

## Product Purpose

A real-time combat encounter manager for D&D 5e. Tracks initiative order, HP, conditions, and stat blocks for all combatants. Pulls live PC character sheets from D&D Beyond. Lets the DM run encounter after encounter without paper or mental overhead. Success means the DM never loses the thread of a fight.

## Brand Personality

Atmospheric. Precise. Present.

The interface should feel like it belongs next to the dice, the map, and the miniatures — not like a SaaS dashboard dragged into the dungeon. Dark, tactile, a little dramatic. No kitsch (no parchment, no scrollwork, no tavern fonts). The drama comes from the data: an active combatant glowing gold, a red HP bar almost empty, a condition chip signaling unconscious. The UI earns its weight through combat clarity, not decorative theme.

## Anti-references

- **Generic SaaS / dashboard** — blue-and-white business tool aesthetic, sidebar nav, metric cards, bland enterprise typography. This is a weapon, not a spreadsheet.
- **Fantasy kitsch** — scrollwork borders, parchment textures, tavern-sign serif fonts. The D&D soul comes from game state, not decoration.
- **Mobile game HUD** — glossy buttons, XP bars, heavy saturation, game-UI chrome. Should feel closer to a specialist instrument than a freemium game.
- **Spreadsheet / table-first** — rows and columns, filter bars. The initiative order is a ranked list with life and death stakes, not a database view.

## Design Principles

1. **Combat readability at a glance** — the active combatant, current HP, and conditions must be scannable from a meter away. Information density earns its place by removing hunting, not by cramming more in.
2. **Atmosphere through state, not decoration** — visual drama (glow, color, emphasis) communicates game state. Red means low HP. Gold means it's your turn. The interface looks intense because the encounter is intense, not because of applied theming.
3. **Peripheral vision design** — key state should read without focus. The DM should be able to glance at the tracker mid-conversation and instantly re-orient.
4. **Tactile feedback** — actions should feel real. HP changes, conditions toggled, turn advancing — each should have a small but satisfying physical response.
5. **The table doesn't wait** — every interaction should resolve in one deliberate move. No confirmation dialogs for normal actions. No friction between intent and execution.

## Accessibility & Inclusion

WCAG AA minimum. High contrast is mandatory — this is used in dim ambient lighting (gaming tables often have low overhead light). No reliance on color alone to convey state; always pair with icon or label. Reduced-motion users get instant transitions.
